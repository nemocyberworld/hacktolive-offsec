Red Teaming Labs

Domain: Red Teaming • Focus: Adversary Emulation, Active Directory, & Attack Simulation • Last Updated: 2025-09-05

Summary

This collection focuses on hands-on red teaming resources: adversary emulation frameworks, Active Directory attack labs, and realistic environments for practicing lateral movement, persistence, and detection evasion. Use these labs ethically, in isolated environments, and with explicit authorization.

Available Labs

  • DetectionLab – Turn-key Windows/AD lab to generate telemetry and test detections.
  • Vulnerable Active Directory – Intentionally insecure AD for practicing common attack paths.
  • Vulnlab – Realistic red team style boxes and scenarios (freemium).
  • FLARE VM – Windows RE/analysis toolkit VM useful for payload dev & analysis.
  • Commando VM – Windows pentest/red team VM preloaded with offensive tooling.
  • HackSys Extreme Vulnerable Driver – Kernel exploit dev playground (Windows).
  • UnderTheWire – Windows/PowerShell CTFs to sharpen operator fundamentals.
  • HStrike – Red team training scenarios and exercises (freemium).
  • MITRE CALDERA – Open-source adversary emulation platform mapped to ATT&CK.
  • Atomic Red Team – Small, testable ATT&CK-mapped techniques for validation.
  • Invoke-AtomicRedTeam – PowerShell runner for Atomic tests (great for Windows/AD).
  • Stratus Red Team – Cloud (AWS/GCP/Azure) adversary emulation of real techniques.
  • Infection Monkey – Open-source breach & attack simulation across networks and clouds.
  • PurpleSharp – Adversary simulation for Windows/AD using .NET (purple teaming).
  • GOAD (Game of Active Directory) – Automated multi-forest AD attack lab with Terraform/Ansible.
  • BadBlood – Quickly seeds AD with users, groups, and objects for realistic attack paths.
  • Splunk Attack Range – Build an attackable lab + detections for purple teaming (IaC).
  • CloudGoat – Vulnerable-by-design AWS scenarios for red team practice.
  • AzureGoat – Intentionally vulnerable Azure environment for attack simulation.
  • GCPGoat – Deliberately insecure GCP project for cloud red teaming.
  • Red Team Automation (RTA) – Minimal ATT&CK-mapped behaviors for detection validation.
