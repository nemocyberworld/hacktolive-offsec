Container Labs

Domain: Container • Focus: Vulnerable Docker & Kubernetes Lab Environments • Last Updated: 2025-09-05

Summary

Container labs offer hands-on learning environments using Docker and Kubernetes to practice exploitation, misconfiguration discovery, privilege escalation, and secure hardening techniques. These labs are invaluable for both offensive and defensive container security learning through immersive, reproducible setups.

Available Labs

  • Vulnerable Docker Labs – A curated collection of intentionally vulnerable Docker images for exploitation practice.
  • Damn Vulnerable Docker DVWA – DVWA packaged as a Docker container to simplify web app vulnerability testing.
  • Juice Shop Docker – OWASP Juice Shop delivered via Docker for quick and easy web app vulnerability exploration.
  • Offensive-Pentesting-Lab – Multi-service Docker images (FTP, DNS, Web, SMB, etc.) designed to simulate vulnerable services in a sandbox.
  • Vulhub – Vast library of pre-built vulnerable Docker Compose environments for practicing known CVE exploits.
  • Kubernetes Goat – A full Kubernetes cluster intentionally designed with realistic misconfigurations and attack paths for K8s security training.
  • ExploitMe - Vulnerable Containers – Small Docker-based labs focused on privilege escalation CVEs and SUID exploits.
  • MBPTL (Most Basic Pentesting Lab) – Beginner-friendly Docker-based pentesting lab that includes a range of classic attack vectors and exploitation scenarios.
