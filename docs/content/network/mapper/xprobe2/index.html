Xprobe2

Domain: Network Reconnaissance • Focus: Active OS Fingerprinting • Last Updated: 2025-09-04

Summary

Xprobe2 is an active OS fingerprinting tool that sends crafted probes (TCP, UDP, ICMP) to identify the operating system of a remote host. It works similarly to Nmap’s OS detection but is lighter and modular, offering customizable probes and analysis. Useful for penetration testing and reconnaissance, Xprobe2 helps security professionals validate target systems before exploitation or risk assessment.

Steps

  1. Prepare the target host or network segment for fingerprinting.
  2. Select probe type (TCP/UDP/ICMP) and relevant options.
  3. Run Xprobe2 with chosen modules and capture responses.
  4. Analyze the OS fingerprint results and document findings.

Commands


# General Options
xprobe2 -v target                       # Be verbose in output
xprobe2 -r target                       # Show traceroute to the target
xprobe2 -h                              # Print help menu
xprobe2 -o results.log target           # Save output to a logfile

# Probes & Ports
xprobe2 -p tcp:23:open target           # Probe TCP port 23 marked as open
xprobe2 -p udp:53:closed target         # Probe UDP port 53 marked as closed
xprobe2 -T21-23,53,110 target           # TCP port scan for multiple ports
xprobe2 -U53,161 target                 # UDP port scan for ports 53 and 161

# Configuration & Modules
xprobe2 -c /path/to/config target       # Use a custom configuration file
xprobe2 -D 2 target                     # Disable module number 2
xprobe2 -M 3 target                     # Enable module number 3
xprobe2 -L                              # List all available modules

# Timing & Debugging
xprobe2 -t 5 target                     # Set receive timeout to 5 seconds
xprobe2 -s 200 target                   # Set packet send delay to 200 ms
xprobe2 -d 2 target                     # Set debug level to 2
xprobe2 -f target                       # Force fixed round-trip time

# Fingerprint Generation
xprobe2 -F -o sig.txt target            # Generate OS signature and save to file
xprobe2 -X -o output.xml target         # Save results as XML to file
xprobe2 -B target                       # Use TCP handshake module to guess open port

# Advanced Analysis
xprobe2 -A -T80,443 target              # Detect suspicious traffic (firewalls, proxies, NIDS)

# Example Usage
xprobe2 -v -p tcp:80 192.168.1.10       # Verbose fingerprint on TCP/80
xprobe2 -T22-25,80,443 10.0.0.5         # TCP portscan on SSH, SMTP, HTTP, HTTPS

Best Flag Combinations (Cheat Ready)


# Quick OS fingerprinting (default TCP probe)
xprobe2 -v 192.168.1.10                  # Verbose fingerprinting on target

# TCP fingerprint scan on common ports
xprobe2 -T21-23,80,443 10.0.0.5          # Scan FTP, Telnet, HTTP, HTTPS for OS hints

# UDP fingerprint scan for DNS & SNMP
xprobe2 -U53,161 192.168.1.20            # Probe UDP 53 (DNS) and 161 (SNMP)

# Save results in XML for automation
xprobe2 -X -o results.xml 192.168.1.30   # XML report for later parsing

# Generate OS signature for target
xprobe2 -F -o sig.txt 192.168.1.15       # Create custom OS fingerprint signature

# Traceroute + fingerprint in one go
xprobe2 -r -v 192.168.1.40               # Show route + verbose OS detection

# Aggressive analysis for suspicious traffic
xprobe2 -A -T80,443 192.168.1.50         # Check proxies/firewalls on HTTP/HTTPS

# Debugging OS detection
xprobe2 -d 3 -p tcp:22:open 10.0.0.25    # Debug level 3, SSH port probe

# Faster timing with fixed RTT
xprobe2 -f -t 2 -s 100 192.168.1.100     # Force RTT=2s, 100ms send delay

# Mixed scan with TCP + UDP probes
xprobe2 -T80,443 -U53 192.168.1.200      # Combine TCP web ports + DNS probe
