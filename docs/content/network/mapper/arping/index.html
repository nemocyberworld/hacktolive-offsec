arping — ARP Ping Utility

Domain: Network • Focus: ARP Probing, Host Discovery & Duplicate IP Detection • Last Updated: 2025-09-04

Summary

arping is a lightweight utility used to send ARP requests to a host and listen for replies. It is primarily used for host discovery, detecting duplicate IP addresses, and updating neighbor ARP tables. Unlike ICMP ping, arping operates at the data link layer, making it effective even when ICMP is blocked. It is often used in offensive security for stealthy host discovery, local network reconnaissance, and verification of IP/MAC mappings.

Steps

  1. Select the network interface to use (e.g., eth0, wlan0).
  2. Choose the mode (normal ARP ping, duplicate detection, unsolicited update).
  3. Run the arping command with appropriate flags for timing, packet count, or source spoofing.
  4. Analyze responses to confirm host availability, detect conflicts, or validate ARP table updates.

Commands


# General Options
arping -f 192.168.1.1                          # Quit after first reply
arping -q 192.168.1.1                          # Quiet mode, suppress output
arping -b 192.168.1.1                          # Keep broadcasting, no unicast
arping -D 192.168.1.50                         # Duplicate address detection (DAD)
arping -U 192.168.1.50                         # Unsolicited ARP: update neighbors (gratuitous ARP)
arping -A 192.168.1.50                         # ARP answer mode: reply update
arping -V                                      # Print version and exit

# Packet Control
arping -c 5 192.168.1.1                        # Send 5 ARP packets then stop
arping -w 3 192.168.1.1                        # Wait max 3 seconds for replies
arping -i 0.5 192.168.1.1                      # Interval of 0.5s between packets

# Interface and Addressing
arping -I eth0 192.168.1.1                     # Use eth0 interface
arping -s 192.168.1.50 192.168.1.1             # Spoof source IP in ARP request

# Example Usage
arping -I eth0 -c 3 192.168.1.1                # Quick 3 ARP probes on eth0
arping -D -I eth0 192.168.1.50                 # Check if 192.168.1.50 is already in use
arping -U -I eth0 192.168.1.50                 # Announce IP change to LAN
arping -A -I eth0 -s 192.168.1.50 192.168.1.1  # ARP reply with spoofed source IP
arping -q -c 1 192.168.1.1                     # Quiet single probe for scripting

Best Flag Combinations (Cheat Ready)


# Quick Host Discovery
arping -I eth0 -c 3 192.168.1.1
   → Send 3 ARP requests, fast availability check

# Stealthy Probe (quiet + single packet)
arping -q -c 1 -I eth0 192.168.1.1
   → Silent one-shot ARP probe (for scripting / automation)

# Detect Duplicate IP (DAD mode)
arping -D -I eth0 192.168.1.50
   → Verify if IP 192.168.1.50 is already taken in the LAN

# Gratuitous ARP Update (announce yourself)
arping -U -I eth0 192.168.1.50
   → Inform neighbors that 192.168.1.50 belongs to your MAC

# ARP Spoof Test (fake source IP)
arping -A -I eth0 -s 192.168.1.50 192.168.1.1
   → Send forged ARP reply to poison/update neighbor cache

# Continuous Broadcast Mode
arping -b -I eth0 192.168.1.1
   → Keep broadcasting ARP requests instead of unicast

# Faster Timing
arping -i 0.2 -c 10 -I eth0 192.168.1.1
   → Send 10 ARP packets at 200ms intervals for reliability
