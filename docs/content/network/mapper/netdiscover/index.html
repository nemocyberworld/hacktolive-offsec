Netdiscover

Domain: Network • Focus: ARP-based Host Discovery • Last Updated: 2025-09-04

Summary

Netdiscover is a lightweight active/passive ARP reconnaissance tool for network discovery. It is especially useful in environments without DHCP servers or when Nmap and other scanners are too noisy. Netdiscover listens for ARP requests or actively probes ranges to quickly map live hosts, their MAC addresses, and associated vendors. It is commonly used in penetration testing, wireless assessments, and network troubleshooting.

Steps

  1. Install Netdiscover (commonly included in Kali Linux and Parrot OS).
  2. Identify target subnet or let Netdiscover auto-detect interfaces.
  3. Run in passive mode (listening) or active mode (sending ARP requests).
  4. Adjust timing, retries, and range according to the environment.
  5. Document live hosts, MAC addresses, and vendors for further scanning.

Commands


# Interface & Targets
-i device                  # Specify network interface
netdiscover -i eth0        # Scan using eth0 interface

-r range                   # Scan a specific CIDR range
netdiscover -r 192.168.1.0/24   # Scan all hosts in 192.168.1.0/24

-l file                    # Scan ranges from file
netdiscover -l ranges.txt       # Load subnets from ranges.txt

-p                         # Passive mode (no packets sent, sniff only)
netdiscover -i wlan0 -p         # Listen for ARP on wlan0 only


# MAC & Filters
-m file                    # Known MAC/hostname mapping file
netdiscover -m maclist.txt      # Use predefined MAC mapping

-F filter                  # Custom pcap filter (default "arp")
netdiscover -F "arp or rarp"    # Capture ARP + RARP packets


# Timing & Performance
-s time                    # Sleep time (ms) between ARPs
netdiscover -s 50 -r 192.168.1.0/24   # 50ms delay between requests

-c count                   # Send each ARP request multiple times
netdiscover -c 3 -r 192.168.1.0/24    # 3 retries per host

-n node                    # Set last octet of source IP (2–253)
netdiscover -n 10 -r 192.168.1.0/24   # Start with .10 as ARP sender

-d                         # Ignore local config files
netdiscover -d -r 192.168.1.0/24      # Fresh scan, no configs used

-R                         # Assume root/caps without checks
netdiscover -R -r 192.168.1.0/24      # Skip privilege validation

-f                         # Fast mode (quicker ARP sweep)
netdiscover -f -r 192.168.1.0/24      # Fast scan through subnet


# Output Modes
-P                         # Parseable output, stop after scan
netdiscover -P -r 192.168.1.0/24      # Print machine-parseable results

-L                         # Like -P but continue listening
netdiscover -L -r 192.168.1.0/24      # Keep ARP sniffing after active scan

-N                         # Suppress header (only with -P or -L)
netdiscover -PN -r 192.168.1.0/24     # Parseable output without headers

-S                         # Suppress sleep (hardcore mode)
netdiscover -S -r 192.168.1.0/24      # Ultra-fast ARP flooding

Best Flag Combinations (Cheat Ready)


# Quick LAN sweep (fast default scan)
netdiscover -r 192.168.1.0/24 -f            # Fast ARP scan of /24 subnet

# Full /16 subnet mapping (slower, thorough)
netdiscover -r 10.0.0.0/16 -c 5 -s 50       # Retry 5 times, 50ms delay

# Passive sniffing (stealth mode)
netdiscover -i wlan0 -p                     # Listen on wlan0, no packets sent

# Continuous monitoring (active + passive)
netdiscover -L -r 192.168.1.0/24            # Scan and keep listening for new hosts

# Parseable output for scripting
netdiscover -P -r 192.168.1.0/24            # Machine-readable ARP results
netdiscover -PN -r 192.168.1.0/24           # Parseable results without headers

# Custom MAC vendor mapping
netdiscover -m maclist.txt -r 192.168.1.0/24 # Use custom MAC-to-vendor file

# Hardcore mode (maximum speed)
netdiscover -S -r 192.168.1.0/24            # Suppress sleep, rapid ARP floods

# Custom filter for extended reconnaissance
netdiscover -F "arp or rarp or ip" -r 192.168.1.0/24  # Capture more than ARP

# Mixed retries + custom delays (resilient scan)
netdiscover -c 3 -s 20 -r 192.168.1.0/24    # 3 ARPs per host, 20ms apart

# Large-range sweep with config ignored
netdiscover -d -r 172.16.0.0/12 -f          # Fast scan, ignore configs
