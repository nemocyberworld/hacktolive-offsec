Nping

Domain: Network • Focus: Packet Generation, Timing, and Response Analysis • Last Updated: 2025-09-04

Summary

Nping is a flexible network packet generation and response analysis tool, distributed with the Nmap suite. It allows testers to craft custom TCP, UDP, ICMP, and ARP packets to evaluate firewall rules, latency, packet loss, and IDS/IPS evasion. Nping is invaluable for performance benchmarking, security testing, and troubleshooting connectivity — often functioning as a more advanced alternative to ping with precise control over payloads and timing.

Steps

  1. Install the Nmap package (Nping comes bundled with it).
  2. Select target host(s) or subnet for probing.
  3. Choose protocol (ICMP, TCP, UDP, ARP) and customize flags/payloads.
  4. Adjust timing, retries, and output mode to fit the scenario.
  5. Interpret results to measure latency, detect filtering, or fingerprint defenses.

Commands


# Target Specification
nping scanme.nmap.org                 # Hostname target
nping microsoft.com/24                 # CIDR target
nping 192.168.0.1;10.0.*.1-24          # Range/wildcard target

# Probe Modes
--tcp-connect                          # Unprivileged TCP connect mode
nping --tcp-connect -p 80 example.com  # Full TCP connect probe

--tcp                                  # TCP probe mode
nping --tcp -p 22 example.com          # Send TCP SYN to port 22

--udp                                  # UDP probe mode
nping --udp -p 53 example.com          # UDP probe to DNS

--icmp                                 # ICMP probe mode
nping --icmp -c 5 8.8.8.8              # Send 5 ICMP echo requests

--arp                                  # ARP probe mode
nping --arp 192.168.1.0/24             # ARP scan on LAN

--tr, --traceroute                     # Traceroute mode
nping --tr --icmp example.com          # ICMP traceroute


# TCP Connect Mode Options
-p, --dest-port                 # Destination port(s)
nping --tcp-connect -p 80,443 example.com

-g, --source-port                # Source port
nping --tcp-connect -g 12345 -p 80 example.com


# TCP Probe Mode Options
--seq                             # Set sequence number
nping --tcp -p 80 --seq 1000 example.com

--flags                          # Set TCP flags
nping --tcp -p 80 --flags SYN,ACK example.com

--ack                             # Set ACK number
nping --tcp -p 80 --ack 12345 example.com

--win                            # Window size
nping --tcp -p 80 --win 2048 example.com

--badsum                               # Invalid checksum
nping --tcp -p 80 --badsum example.com


# UDP Probe Mode Options
nping --udp -p 53 --badsum example.com # UDP with invalid checksum


# ICMP Probe Mode Options
--icmp-type                      # ICMP type
nping --icmp --icmp-type 8 8.8.8.8     # Echo request

--icmp-code                      # ICMP code
nping --icmp --icmp-type 3 --icmp-code 1 8.8.8.8

--icmp-id                          # Identifier
nping --icmp --icmp-id 999 8.8.8.8

--icmp-seq                          # Sequence number
nping --icmp --icmp-seq 5 8.8.8.8

--icmp-redirect-addr             # Redirect address
nping --icmp --icmp-type redirect --icmp-redirect-addr 192.168.1.1 target

--icmp-param-pointer              # Parameter problem pointer
nping --icmp --icmp-type param-problem --icmp-param-pointer 4 target

--icmp-advert-lifetime           # Router adv. lifetime
nping --icmp --icmp-type router-advert --icmp-advert-lifetime 600 target

--icmp-advert-entry           # Router adv. entry
nping --icmp --icmp-type router-advert --icmp-advert-entry 192.168.1.1,100 target

--icmp-orig-time                   # Originate timestamp
nping --icmp --icmp-type timestamp --icmp-orig-time 123456 target

--icmp-recv-time                   # Receive timestamp
nping --icmp --icmp-type timestamp --icmp-recv-time 123789 target

--icmp-trans-time                  # Transmit timestamp
nping --icmp --icmp-type timestamp --icmp-trans-time 123890 target


# ARP/RARP Probe Mode Options
--arp-type                       # ARP, ARP-reply, RARP, etc.
nping --arp --arp-type ARP 192.168.1.1

--arp-sender-mac                  # Sender MAC
nping --arp --arp-sender-mac 00:11:22:33:44:55 target

--arp-sender-ip                    # Sender IP
nping --arp --arp-sender-ip 192.168.1.100 target

--arp-target-mac                  # Target MAC
nping --arp --arp-target-mac ff:ff:ff:ff:ff:ff target

--arp-target-ip                    # Target IP
nping --arp --arp-target-ip 192.168.1.1 target


# IPv4 Options
-S, --source-ip                    # Source IP address
nping --tcp -p 80 -S 192.168.1.50 target

--dest-ip                          # Destination IP address
nping --tcp --dest-ip 10.0.0.5 -p 80

--tos                             # Type of service
nping --tcp --tos 16 -p 80 target

--id                               # Identification field
nping --tcp --id 1234 target

--df                                   # Don't Fragment flag
nping --tcp --df -p 80 target

--mf                                   # More Fragments flag
nping --tcp --mf -p 80 target

--evil                                 # Evil flag
nping --tcp --evil -p 80 target

--ttl                            # Time to live
nping --tcp --ttl 2 target

--badsum-ip                            # Invalid IP checksum
nping --tcp --badsum-ip target

--ip-options                     # Set IP options
nping --tcp --ip-options R 192.168.1.1 target

--mtu                            # Set MTU, force fragmentation
nping --tcp --mtu 100 target


# IPv6 Options
-6, --IPv6                             # Enable IPv6
nping -6 --tcp -p 80 ipv6.example.com

--dest-ip                          # Destination IPv6 address
nping -6 --tcp --dest-ip fe80::1 -p 80

--hop-limit                         # Hop limit
nping -6 --tcp --hop-limit 4 target

--traffic-class                 # IPv6 traffic class
nping -6 --tcp --traffic-class 10 target

--flow

Best Flag Combinations (Cheat Ready)


# Quick ICMP latency test (like ping, but richer)
nping --icmp -c 5 8.8.8.8                   # 5 ICMP echo requests with RTT stats

# TCP SYN handshake probe (stealthy port test)
nping --tcp -p 80 --flags SYN example.com   # SYN-only probe to check port filtering

# TCP connect mode (service availability check)
nping --tcp-connect -p 443 example.com      # Full TCP connect to HTTPS port

# UDP service check
nping --udp -p 53 example.com               # UDP probe against DNS server

# ARP host discovery (LAN only)
nping --arp 192.168.1.0/24                  # Discover live hosts on local subnet

# Traceroute-style probing
nping --tr --icmp example.com               # ICMP-based traceroute
nping --tr --tcp -p 80 example.com          # TCP-based traceroute to port 80

# Firewall/IDS evasion (bad checksum + custom TTL)
nping --tcp -p 80 --flags SYN --badsum --ttl 3 target

# Custom payload injection (testing filters or services)
nping --udp -p 161 --data-string "hello" target
nping --tcp -p 80 --data deadbeef target

# Echo client/server (connectivity + latency measurement)
nping --echo-server "secretpass" -e eth0 -vv   # Start echo server
nping --echo-client "secretpass" echo.nmap.org --tcp -p 80

# High-speed stress test
nping --tcp -p 80 --rate 1000 target         # 1000 packets/sec to port 80

# IPv6 probing
nping -6 --icmp -c 3 ipv6.google.com         # ICMPv6 echo requests

# Combined reliability probe
nping --tcp -p 22 --flags SYN --count 5 --delay 1s target
