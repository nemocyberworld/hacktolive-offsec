ssdp-scanner

Domain: Network Enumeration • Focus: UPnP/SSDP Device Discovery • Last Updated: 2025-09-04

Summary

ssdp-scanner is a reconnaissance tool used to discover devices and services on a network that expose the Simple Service Discovery Protocol (SSDP), typically used by Universal Plug and Play (UPnP). It broadcasts M-SEARCH requests and parses responses to identify devices such as routers, IoT appliances, smart TVs, and printers. Security testers use it to uncover misconfigured or vulnerable UPnP services that can leak information about internal networks or allow exploitation via exposed control endpoints.

Steps

  1. Identify the broadcast/multicast address for SSDP queries (default: 239.255.255.250:1900).
  2. Run ssdp-scanner to send discovery requests on the local subnet.
  3. Parse responses for device details such as server type, location URL, and UPnP services.
  4. Investigate discovered endpoints for misconfigurations or vulnerabilities.

Commands


# Basic Scan
ssdp-scanner                                   # Discover devices via SSDP on default multicast

# Custom Interface
ssdp-scanner -i eth0                           # Specify network interface to use

# Targeted Broadcast
ssdp-scanner -t 192.168.1.255                  # Send queries to a specific broadcast address

# Custom Timeout
ssdp-scanner -T 10                             # Set timeout for responses to 10 seconds

# Verbose Output
ssdp-scanner -v                                # Enable verbose mode for detailed results

# Example Usage
ssdp-scanner                                   # Quick discovery of UPnP/SSDP devices
ssdp-scanner -i wlan0 -T 5                     # Discover devices on Wi-Fi, 5s response timeout

Best Flag Combinations (Cheat Ready)


# Quick discovery on default multicast
ssdp-scanner                                   # Fast scan for all SSDP devices

# Wi-Fi interface scan with timeout
ssdp-scanner -i wlan0 -T 5                     # Use wlan0, wait 5s for responses

# Targeted broadcast scan
ssdp-scanner -t 192.168.1.255 -T 8             # Send M-SEARCH to broadcast, 8s wait

# Verbose enumeration
ssdp-scanner -v                                # Print full SSDP/UPnP response details

# Custom interface + verbose
ssdp-scanner -i eth0 -v                        # Detailed results via eth0 interface

# Focused discovery on IoT subnet
ssdp-scanner -t 10.0.0.255 -i eth1             # Scan IoT VLAN with eth1
