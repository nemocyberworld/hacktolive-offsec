The access control matrix defines who can access which resources and perform which actions in a system. Disclosure of this matrix can expose the full set of permissions and user roles, enabling attackers to map out privileges and identify attack vectors.

1. What Is the Access Control Matrix?

It is a conceptual model representing permissions assigned to users or roles across all resources. It can include data like:

  • User-to-resource access rights
  • Role-based permissions
  • Allowed operations per user or role

2. Why Is Disclosure Dangerous?

  • Attackers learn exactly which accounts have admin or elevated privileges.
  • It reveals sensitive structural information about the system.
  • Facilitates targeted attacks like privilege escalation or account takeover.

3. Common Causes

  • Improper error messages or debug output leaking ACL details.
  • Exposed endpoints returning full permission matrices.
  • Misconfigured API responses or admin panels accessible to unauthorized users.

4. Example Scenario

An API endpoint /api/permissions returns a JSON object like this to an authenticated user:

{
  "users": {
    "alice": ["read", "write", "admin"],
    "bob": ["read", "write"],
    "eve": ["read"]
  }
}

If this endpoint is accessible by low-privilege users or publicly, attackers can map permissions and find targets with elevated rights.

5. Risks and Impacts

  • Targeted social engineering or phishing on high-privilege users.
  • Privilege escalation by focusing attacks on admin accounts.
  • Reconnaissance ease leading to faster exploitation.

6. How to Test

  • Review API responses for permission-related data leakage.
  • Check error messages for sensitive role or permission info.
  • Perform access tests to confirm permission data is restricted.

7. Mitigation Strategies

  • Restrict access to permission data strictly to authorized admin users.
  • Sanitize error messages to avoid leaking ACL details.
  • Audit API endpoints and admin consoles for unintended data exposure.
  • Use role-based access control (RBAC) and principle of least privilege.

Summary

  • Access control matrix disclosure aids attackers in privilege reconnaissance.
  • Keep permission data confidential and only accessible to trusted admins.
  • Test and sanitize all responses and errors to prevent leakage.
