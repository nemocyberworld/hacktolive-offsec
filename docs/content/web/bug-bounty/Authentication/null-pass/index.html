This guide explains the security flaw where login systems accept null or empty passwords, allowing unauthorized access.

1. What Is the Issue?

Some login implementations incorrectly allow authentication if the password field is left empty or null, bypassing proper verification.

2. Risks and Impacts

  • Unauthorized user access without valid credentials.
  • Complete account compromise, including privileged accounts.
  • Potential for automated mass exploitation.

3. How to Test

  • Attempt login with valid usernames and empty password fields.
  • Test null, empty strings, whitespace, or unusual encodings.
  • Observe if access is granted or error messages reveal flawed handling.

4. Prevention Strategies

  • Always validate inputs and reject empty or null passwords before authentication.
  • Enforce strong password policies and input sanitation.
  • Use secure authentication libraries that correctly handle empty inputs.
  • Implement logging and monitoring for suspicious login attempts.

5. Example: Secure Password Check (Python)

def authenticate(username, password):
    if not password:
        return False  # Reject empty/null passwords
    user = find_user(username)
    if user and verify_password(user, password):
        return True
    return False

Summary

  • Empty or null password acceptance is a critical vulnerability.
  • Always sanitize and validate authentication inputs strictly.
  • Test your login flows rigorously against such bypass attempts.
