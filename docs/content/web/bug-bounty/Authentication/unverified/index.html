This guide covers the security risk of allowing users to log in before verifying their email addresses.

1. What Is the Issue?

Allowing login without verifying the email address can lead to unauthorized access, impersonation, and fraudulent account creation.

2. Risks and Impacts

  • Attackers can create accounts with fake or stolen emails and access the application.
  • Potential for phishing and spam through unverified accounts.
  • Compromise of trust in the user base and application integrity.

3. How to Test

  • Register a new account with a valid but unverified email.
  • Attempt to log in without clicking on the verification link.
  • Check if access is granted or restricted.

4. Prevention Strategies

  • Require email verification before allowing login.
  • Send verification emails with unique, time-limited tokens.
  • Implement checks in the authentication flow to enforce verification status.
  • Provide clear user feedback on verification status.

5. Example: Email Verification Check (Python)

def login(user_email, password):
    user = find_user_by_email(user_email)
    if not user:
        return False
    if not user.email_verified:
        return "Email not verified", 403
    if verify_password(user, password):
        return True
    return False

Summary

  • Prevent login before email verification to ensure user authenticity.
  • Use robust email verification mechanisms with token expiry.
  • Test your app for this flaw to protect users and data.
