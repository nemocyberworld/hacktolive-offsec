This guide explains the session fixation vulnerability where attackers fix a user's session ID before login, enabling them to hijack the session after authentication.

1. What Is Session Fixation?

Session fixation occurs when an attacker sets or predicts a valid session ID (session token) and tricks the user into authenticating with that session ID. If the server does not issue a new session ID upon login, the attacker can reuse the same session ID to hijack the authenticated session.

2. Risks and Impacts

  • Unauthorized access to victim accounts.
  • Data theft, privilege escalation, or transaction manipulation.
  • Loss of user trust and potential regulatory penalties.

3. How to Test

  • Try setting a session ID manually (via cookie or URL) before login.
  • Log in using the fixed session ID.
  • Check if the session ID remains the same after authentication.
  • If yes, the application is vulnerable.

4. Prevention Strategies

  • Regenerate a new session ID immediately after login.
  • Invalidate old session IDs on login and logout.
  • Use secure, HttpOnly, and SameSite cookie flags.
  • Implement short session lifetimes and inactivity timeouts.

5. Example: Regenerate Session ID in Flask

from flask import session
from flask_login import login_user

def login(user):
    login_user(user)
    session.regenerate()  # Pseudocode; use flask_session or similar to regenerate session ID

Summary

  • Session fixation is a critical security flaw allowing attackers to hijack sessions.
  • Always regenerate session IDs after authentication to prevent fixation.
  • Use secure cookie attributes and monitor session activity.
